# Command
/prd

## Purpose
Generates a structured Product Requirements Document from a validated problem or opportunity — the artifact every downstream architecture, BOM, and financial model depends on.

## Activated Agents
- CPO-Agent

## Activated Skills
- `01-discovery-skill` — problem, ICP, JTBD
- `04-prd-skill` — structured PRD

## Workflows
- `01-new-product-workflow`
- `02-saas-product-workflow`

## Output
- PRD (Problem, Goals/Non-Goals, Requirements, Success Metrics, Phased Roadmap)

## Example
`/prd Write a PRD for a usage-based billing feature on our SaaS platform.`
