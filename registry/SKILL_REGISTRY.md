# Skill Registry

This registry is the canonical reference for every Skill in FoundryOS. Each Skill compiles a small cluster of Modules (the `122_FoundryOS_Module_Map`-defined atomic units) into one reusable capability, and is owned by exactly one Agent — except `35-npi-manufacturing-skill`, which is intentionally co-owned by COO-Agent and CIO-Agent. "Dependencies" below means the other Skill(s) whose output is the most natural input to this one in a typical end-to-end run, not a hard technical dependency — the Meta-Agent decides per-request whether to actually run them in sequence.

Total: **59 Skills** compiled from **179 Modules**, owned by **10 Agents**.

## Notes

- **Shared ownership:** `35-npi-manufacturing-skill` is the only Skill owned by two Agents. CIO-Agent runs it for the hardware-engineering side of NPI (DFM, EVT/DVT/PVT); COO-Agent runs it for the operations side (manufacturing ramp, supplier qualification, process control). Both copies are kept in sync with the canonical `skills/35-npi-manufacturing-skill/` folder.
- **Cross-cutting Skills** (`14-validation-skill`, `15-framework-library-skill`, `16-artifact-template-skill`, `17-diagram-skill`, `25-prompt-loop-skill`, `26-knowledge-graph-skill`, `28-risk-management-skill`, `40-meta-orchestration-skill`, `59-problem-solving-decision-modeling-skill`) don't sit in one linear pipeline position — they apply on top of whatever artifact another Skill just produced, which is why their "Dependencies" column points outward rather than to one upstream Skill.
- **`59-problem-solving-decision-modeling-skill`** is a reasoning engine, not a document template — it's the discipline other Skills and Agents borrow whenever a request is actually a decision ("should we," "which option," "why is this happening," "is this worth it") rather than a request for a known artifact. It combines with whichever domain Skill(s) the decision touches (e.g. a robotics decision runs it alongside `23-robotics-product-skill`; a pricing decision runs it alongside `07-finance-skill` and `36-pricing-sales-skill`) rather than replacing them. See `../skills/59-problem-solving-decision-modeling-skill/SKILL.md`.
- `41-mechatronics-skill` was added after the original 40-skill map to give hardware-heavy requests (robotics, embedded, mechanical, electronics) one integrated discipline instead of three separately-validated ones. The three original Skills (`32`, `33`, `34`) were kept intact rather than deleted, so existing routing that targets them individually still works.
- **Brand cluster (`42`–`58`):** added in v4 alongside CBO-Agent to give brand strategy, identity, design systems, voice, content, community, and rollout the same first-class treatment the original 41 Skills give product, engineering, and operations. See [`../brand/BRAND_OS.md`](../brand/BRAND_OS.md) for how this cluster connects to every other layer.
- **Naming correction:** the original Brand OS request specified two Skills named `logo-system-skill` (slots 46 and 57). Slot 57 has been renamed `brand-assets-management-skill` to remove the collision — its actual function (asset library structure, file naming, versioning) was always distinct from `46-logo-system-skill`'s function (the logo mark itself), so the rename clarifies scope rather than just resolving a name clash. Flagged in [`../AUDIT_REPORT.md`](../AUDIT_REPORT.md).
